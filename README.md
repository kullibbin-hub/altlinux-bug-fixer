Скрипт предназначен для фикса багов и начальной настройки дистрибутива Altlinux workstation 11.1. Скрипт получает sudo, обновляет систему и ядро, 
затем устанавливает некоторые нужные пакеты и расширения, отключает службу индексатора дисков и фиксит некоторые баги и неудобства, а именно:
1. Долгий запуск центра приложений, решается отключением автообновления в его настройках. При этом все равно обновления будут проверяться и выдаваться сообщение об их наличии.
   https://bugzilla.altlinux.org/show_bug.cgi?id=57455
2. Включение разрешения приложениям из flatpak на доступ к домашнему каталогу. Позволяет перетаскивать файлы мышкой на приложение из окна наутилуса или рабочего стола.
3. Уменьшает кэш записи vm.dirty_bytes и vm.dirty_background_bytes до разумных значений 64 и 16 мБ, после этого индикатор копирования в nautilus показывает
   реальный прогресс копирования, и при копировании на флешку отдает ее по окончании копирования через 2 - 3 сек. До этого кэш по умолчанию равен 10% от ОЗУ и после   завершения копирования,
   по сообщению системы, еще минуту или больше на флешку сбрасывается кэш, что очень некомфортно для пользователя.
   https://bugzilla.altlinux.org/show_bug.cgi?id=57387
4. Фикс ложного включения gsconnect, даже при отключенном расширении, если телефон был сопряжен с компьютером, все равно работает синхронизация.
Скрипт решает эту проблему, прибивая процесс, если gsconnect отключен. https://bugzilla.altlinux.org/show_bug.cgi?id=57205
5. Настройки nautilus по умолчанию не оптимальны и неудобны. Скриптом будут включены опции: папки перед файлами, создание ссылок, удаление помимо корзины,
   и доп. информация о файлах в режиме значков. Также устанавливается пакет nautilus-admin-gtk4. https://bugzilla.altlinux.org/show_bug.cgi?id=57475
6. Фикс появления фантомных устройств в индикаторе съемных дисков в панели. Если флэшку вставить до логина пользователя, в индикаторе появится еще одна фантомная флэшка,
   которая там остается даже после отмонтирования реальной. Предыдущий фикс оказался неэффективным, теперь просто патчится js файл расширения, как предложено в багзилле.
    https://bugzilla.altlinux.org/show_bug.cgi?id=57356

Также, скрипт активирует sudo, обновляет систему, устанавливает epm, который по умолчанию отсутствует, обновляет ядро, и далее устанавливает: 
flatseal, bleachbit, 
extension-manager, synaptic, epmgpi, eepm-play-gui, baobab, 
sushi, luckybackup, qdiskinfo, gearlever, android-tools, 
java-21-openjdk, grub-theme-dark, icon-theme-Papirus, 
icon-theme-Papirus-Dark, pipewire-jack. 

А также расширения gnome shell:
add to desktop, blur my shell, no overview at startup.

По мере того, как разработчики дистрибутива будут фиксить баги, буду убирать из скрипта то, что уже не нужно. Но пока, на конец февраля 2026 г, все актуально.

Использование.
По зеленой кнопке "<>Code" вверху страницы выбрать Download zip, скачается архив. Распаковать в домашний каталог (можно оставить в папке "Загрузки"), в распакованной папке в свойствах скрипта "настройка-после-установки+фиксы.sh" по правому клику включить "Исполняемый как приложение".
Правый клик по скрипту - копировать, открыть терминал, правый клик - вставить. Enter. Желательно чтоб скрипт не находился в папке с пробелами, например, "Рабочий стол" или "мой софт". 
Лучше переместить его в корень домашнего каталога, как уже писал. Также, можно запустить скрипт правым кликом по нему - "запустить как приложение". Скрипт 2 раза запросит ввод пароля, 
первый раз для получения sudo, и еще раз для начала его использования. Все этапы работы выводятся в терминал.

2. Добавлена установка пакета nautilus-admin-gtk4 для получения пункта в меню файлменеджера - "Открыть как администратор".

4. Улучшена корректность работы скрипта при исправления бага с gsconnect.
