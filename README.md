Скрипт altlinux-bug-fixer.sh предназначен для исправления ошибок и начальной настройки дистрибутива Altlinux workstation 11.1 и Starterkit для домашнего использования. Код скрипта разделен на блоки, можно просто удалить любой блок от разделителя до разделителя, чтобы убрать какую-либо функцию и настроить "под себя". В архиве scripts.zip - отдельные функции выделены в отдельные скрипты, можно запустить оттуда только нужные фиксы. Там же есть readme.txt с описанием скриптов.

Скрипт altlinux-bug-fixer.sh получает sudo, обновляет систему, добавляет текущего пользователя в группу dialout для доступа к оборудованию через usb, 
затем устанавливает некоторые нужные пакеты и расширения и фиксит некоторые баги и неудобства, а именно:

1. Долгий запуск центра приложений, решается отключением автообновления в его настройках. При этом все равно обновления будут проверяться и выдаваться сообщение об их наличии.
   https://bugzilla.altlinux.org/show_bug.cgi?id=57455
2. Включение разрешения приложениям из flatpak на доступ к домашнему каталогу. Позволяет перетаскивать файлы мышкой на приложение из окна наутилуса или рабочего стола.
3. Уменьшает кэш записи vm.dirty_bytes и vm.dirty_background_bytes до разумных значений 64 и 16 мБ, после этого индикатор копирования в nautilus показывает
   реальный прогресс копирования, и при копировании на флешку отдает ее по окончании копирования через 2 - 3 сек. До этого кэш по умолчанию был равен 10% от ОЗУ и после   завершения копирования,
   по сообщению системы, еще минуту или больше на флешку сбрасывается кэш, что очень некомфортно для пользователя.
   https://bugzilla.altlinux.org/show_bug.cgi?id=57387
4. Фикс ложного включения gsconnect, даже при отключенном расширении, если телефон был сопряжен с компьютером, все равно работает синхронизация.
Скрипт решает эту проблему, прибивая процесс через 10 сек после логина, если gsconnect отключен. https://bugzilla.altlinux.org/show_bug.cgi?id=57205
5. Настройки nautilus по умолчанию не оптимальны и неудобны. Скриптом будут включены опции: папки перед файлами, создание ссылок, удаление помимо корзины,
   и доп. информация о файлах в режиме значков. Также устанавливается пакет nautilus-admin-gtk4. https://bugzilla.altlinux.org/show_bug.cgi?id=57475
6. Фикс появления фантомных устройств в индикаторе съемных дисков в панели. Если флэшку вставить до логина пользователя, в индикаторе появится еще одна фантомная флэшка,
   которая там остается даже после отмонтирования реальной. Патчится js файл расширения, как предложено в багзилле.
    https://bugzilla.altlinux.org/show_bug.cgi?id=57356

=================  Использование:  ======================

По зеленой кнопке "<>Code" вверху страницы выбрать Download zip, скачается архив. Распаковать в домашний каталог (можно оставить в папке "Загрузки"), в распакованной папке в свойствах скрипта "altlinux-bug-fixer.sh" по правому клику включить "Исполняемый как приложение".
Правый клик по скрипту - копировать, открыть терминал, правый клик - вставить. Enter. Желательно чтоб скрипт не находился в папке с пробелами, например, "Рабочий стол" или "мой софт". 
Лучше переместить его в корень домашнего каталога, или оставить в папке "Звгрузки", как уже писал. 
Также, можно запустить скрипт правым кликом по нему - "запустить как приложение". Скрипт 2 раза запросит ввод пароля, 
первый раз для получения sudo, и еще раз для начала его использования. Все этапы работы выводятся в терминал.

После отработки скрипта очень желательно перезагрузить компьютер, а затем зайти в настройки Gnome - Приложения - Приложения по умолчанию, 
и проверить привязку приложений к типам файлов, при необходимости поправить.

Если у вас Sterterkit - теперь легко можно переключиться в панельный режим. Запустить tuner и на первой вкладке будет выбор раскладки панелей. В менеджере расширений можно выбрать нужные расширения "по вкусу".

Насчет службы localsearch-3, которая индексирует СОДЕРЖИМОЕ файлов, решение ее не трогать - неоднозначное. При наличии большого числа MIDI-файлов она может серьезно нагружать жесткий диск, и подвисать при попытке ее остановить. Проверить можно просто. 
В терминале:

systemctl --user stop localsearch-3

Если процесс остановки занимает длительное время, и не нужен поиск по содержимому файлов,
можно ее вообще отключить для текущего пользователя. В терминале:

systemctl --user mask localsearch-3

systemctl --user stop localsearch-3

и проверить задержку при перезагрузке. Если в процессе перезагрузки нажать ESC, то можно
видеть подробно лог происходящего.

Включить localsearch-3 обратно:

systemctl --user unmask localsearch-3

systemctl --user start localsearch-3

В дальнейшем есть планы сделать GUI для выбора опций для оптимизации.

Также, у кого после последних обновлений стал очень долго выключаться компьютер, проблема может быть в расширении Gtk4 Desktop Icons. Просто через менеджер расширений, который заодно устанавливает данный скрипт, сначала отключить Gtk4 Desktop Icons, и затем через вкладку "Обзор" установить Desktop Icons NG (без GTK4). Зависания при выключении прекратятся. В данное время баг в стадии исправления, поэтому не добавлял установку этого расширения.

Если система Starterkit, и не установлены расширения для нижней панели и значков на столе, но есть желание это установить - можно это сделать в терминале одной командой:

sudo apt-get install -y gnome-shell-extension-dash-to-panel gnome-shell-extension-gtk4-desktop-icons-ng 

Можно установить тему для значков papirus-remix с цветными папками, выбрать тему можно будет через Тюнер - Внешний вид - Темы - Значки. Установка этой темы занимает довольно значительное время, при желании можно это сделать самостоятельно 
через терминал:

sudo apt-get install -y papirus-remix-icon-theme

Установка flatpak и flathub - для установки  огромного числа приложений с зарубежного международного репозитория flathub:

sudo apt-get install -y flatpak firsttime-flatpak-mask-openh264 flatpak-fix-openh264+stplr-test-flatpak-fix-openh264 flatpak-repo-flathub

Установка репозитория aides и к нему stplr - еще приложения из российского репозитория:

sudo apt-get install -y stplr stplr-repo-aides gnome-software-plugin-stplr
